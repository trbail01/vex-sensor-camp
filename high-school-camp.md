# High School Camp

## Slightly More Advanced Goals

- Read sensor values instead of only treating sensors as on/off switches
- Use thresholds, loops, and variables in robot code
- Debug behavior by comparing readings to robot actions
- Build a short autonomous routine that reacts to the environment

## 2-Hour Schedule

| Time | Activity |
| --- | --- |
| 0:00-0:10 | Welcome, robot tour, and sensor overview |
| 0:10-0:20 | Live sensor values and threshold mini lesson |

| 0:35-0:55 | Mission 2: Pressure Sensor Start |

## Vocabulary

| Word | Meaning |
| --- | --- |
| Threshold | A value that tells the program when to act |
| Loop | Code that repeats |
| Variable | A named place to store a value or state |
| Debugging | Finding the reason code or hardware is not behaving as expected |
| Autonomous | The robot acts on its own based on code |
| Sensor value | The number or reading a sensor returns |

## Instructor Notes

- Ask students to explain their thresholds and not just copy one from another team.
- Encourage them to print or view live sensor readings before changing code.
- Push for variables with meaning, such as startPressed, grabDistance, or lineThreshold.
- Keep the final challenge open enough for student problem solving, but narrow enough to finish in 2 hours.

## Reflection Questions

- What sensor reading mattered most in your program?
- How did your threshold affect robot behavior?
- Where did a variable make your code easier to understand?
- What debugging step helped the most?
- How is this similar to a smart home or other automated system?

## Optional IoT Connection

If time allows, compare the robot to a Home Assistant automation. A sensor in a room can play the same role as a robot sensor: detect input, run logic, and trigger output.
