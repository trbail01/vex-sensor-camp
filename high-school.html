High School Camp

This version of camp digs into sensor values, thresholds, loops, variables, and debugging. Students still use the same pre-built VEX claw robots, but the programming decisions are more open-ended and analytical.

Camp goals

  • Interpret live sensor values instead of only using simple on/off triggers.
  • Use thresholds, loops, variables, and if/else logic.
  • Debug behavior by comparing sensor readings to robot actions.
  • Build a short autonomous routine that reacts to the environment.

Sensor values and thresholds

Sensors often return numbers. A threshold is a value your program uses to decide when to act. Example: if the distance sensor reads less than 100 mm, the object is close enough to grab.

Mission 1: Pressure Sensor Mode Toggle or Start/Stop

Use the back pressure sensor to change robot behavior. One press might start the robot, while another press could stop it or switch modes.

Mission 2: Range Sensor Claw Grabber

Read distance values continuously. When the value crosses your chosen threshold, stop driving and close the claw around the object.

Mission 3: Optical Sensor Surface Detector

Use brightness or color data from the optical sensor to detect a dark zone, tape line, or target surface before the robot changes behavior.

Final Challenge: Smart Claw Autonomous Mission

Design a short autonomous program that starts correctly, reacts to the environment, picks up an item, and finishes in a target zone.

Challenge requirements

  • Use at least two sensors.
  • Include one loop.
  • Include one if/else decision.
  • Use one variable to store a value or state.

Optional IoT connection

If time allows, compare robot logic to a Home Assistant automation. For example, a Zigbee motion sensor can act like an input, an automation rule is the logic, and a smart bulb or switch is the output.

Reflection questions

  • How did you choose your sensor thresholds?
  • What data helped you debug the robot when it missed an object or stopped too soon?
  • Why is it useful to store values in variables during autonomous behavior?
  • What real-world systems depend on sensors, data, and automated decisions?
