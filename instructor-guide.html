Planning notes for facilitators

Instructor Guide

Use this page as a quick setup and delivery guide for a 2-hour VEX Sensor Camp session. Adjust pacing based on group age, confidence, and the number of working robots available.

Materials list

  • Pre-built VEX robots with claws
  • Pressure, optical, and range sensors installed
  • Laptops with required VEX software already installed
  • Charging cables, batteries, and spares
  • Small grab objects, bins, tape lines, and target zones
  • Optional: Raspberry Pi, Home Assistant, Aqara sensors, smart bulbs, smart switches

Room setup

  • One programming station per team with room to test on the floor or tables.
  • Mark a practice lane, line-follow area, and pickup zone with tape.
  • Keep a demo robot at the front for fast whole-group explanations.
  • Set up a charging area and a spare-parts area.

Suggested group size

Pairs work best. Groups of three can work if robot count is limited, but assign clear roles such as driver, coder, and tester.

Timing guide

Keep instruction bursts short. Aim for 5-10 minute mini lessons followed by hands-on testing so students stay active and can quickly connect code to robot behavior.

Setup checklist

  • Charge all robots and controller batteries
  • Open the programming software on each laptop
  • Lay out challenge objects and taped paths
  • Prepare one working demo program per mission

Robot checklist

  • Drive motors respond correctly
  • Claw opens and closes
  • Pressure sensor is firmly mounted and readable
  • Distance sensor faces forward near the claw
  • Optical sensor points toward the floor

Laptop/software checklist

  • Correct software version installed
  • Robots connect reliably by cable or wireless link
  • Students can download and run sample code
  • Sensor monitor or value display is available

Troubleshooting tips

  • If a sensor seems wrong, check port assignments and cable connections first.
  • If the claw misses objects, slow the drive speed before changing logic.
  • If the optical sensor is inconsistent, recalibrate on the actual floor surface.
  • Have students read live values before guessing at fixes.

Backup plan

  • If one sensor fails, turn that mission into a teacher demo and let students focus on the remaining two sensors.
  • If a robot stops working, rotate teams through a shared test station while others plan logic on paper.
  • Keep printed pseudocode cards ready so progress can continue without hardware.

Middle school vs high school delivery

  • Middle school: model each mission first, keep language simple, and celebrate partial success.
  • High school: ask students to justify thresholds, compare readings, and explain debugging steps.
  • For both groups, keep testing frequent and visible.

Optional Home Assistant / Zigbee demo notes

A short demo can connect robotics to everyday automation. Show a motion or door sensor in Home Assistant, then highlight the same pattern students use in code: input, logic, output. A smart bulb or switch makes the result easy to see.
