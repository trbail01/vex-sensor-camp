# VEX Sensor Camp

A hands-on robotics camp where students use VEX robots to learn how sensors help robots sense, decide, and act.

This repository is organized as Markdown documentation so the main GitHub page stays useful and readable without a website framework.

## Camp Summary

VEX Sensor Camp is a 2-hour robotics experience built around short missions. Students use a pre-built VEX robot with a claw, a pressure sensor on the back, an optical sensor on the bottom, and a range/distance sensor on the front near the claw. The core lesson is simple: sensors provide input, code applies logic, and the robot produces output.

## Audience

- Middle school campers who need a guided introduction
- High school campers who can handle more depth and debugging
- Instructors who want a quick, practical camp plan

## Robots and Sensors

- Driving VEX robot, already built and ready to test
- Claw mechanism for pickup challenges
- Pressure sensor on the back
- Optical sensor on the bottom
- Range/distance sensor on the front near the claw

## Learning Model

| Step | What it means | Camp example |
| --- | --- | --- |
| Input | The robot senses something | A button is pressed or an object gets close |
| Logic | The code decides what to do | If/then rules, thresholds, loops, variables |
| Output | The robot acts | Drive, stop, turn, or close the claw |

## Quick Start for Instructors

1. Read [camp-overview.md](camp-overview.md) to get the big picture.
2. Skim [instructor-guide.md](instructor-guide.md) for setup and backup plans.
3. Pick the camp path you need: [middle-school-camp.md](middle-school-camp.md) or [high-school-camp.md](high-school-camp.md).
4. Open [activities/README.md](activities/README.md) and run the missions in order.
5. Print the handouts from [handouts/README.md](handouts/README.md) if you want paper copies.

Use whichever VEX software your program already uses (for example, VEXcode IQ or VEXcode V5). The pseudocode in this repo is intentionally generic so you can adapt it to blocks or text.

## Middle School and High School Paths

The middle school and high school versions use the same mission structure, but the high school version goes deeper. Middle school focuses on basic sensor ideas, simple if/then logic, and trial and error. High school uses the same missions but adds thresholds, loops, variables, sensor value debugging, and more autonomous decision-making.

## Notes

- Keep the missions short and hands-on.
- Use the same robot language throughout the camp: input, logic, output.
- Link students back to real-world sensors whenever possible.
