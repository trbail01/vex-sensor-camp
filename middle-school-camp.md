# Middle School Camp

## Simple Goals

- Learn what a sensor is
- Use if/then code to make a robot react
- Practice trial and error without getting stuck for too long
- Finish a set of short missions with a claw robot

## Vocabulary

| Word | Simple meaning |
| --- | --- |
| Sensor | A part that helps a robot notice something |
| Input | Information the robot receives |
| Output | What the robot does |
| Logic | The code that tells the robot what to do |
| Condition | A test in the code, like a button being pressed |
| Trial and error | Testing, changing, and trying again |
| Debugging | Finding and fixing problems in code |

## Instructor Notes

- Keep explanations short and show each idea with the robot.
- Use the words input, logic, and output every time students ask what something does.
- Let students see sensor values if the software makes that easy, but do not turn the lesson into a lecture.
- Celebrate partial success. A robot that reacts the wrong way still teaches something useful.

## Reflection Questions

- Which sensor was easiest to understand?
- What did the robot notice that you did not notice at first?
- What changed when you adjusted the code and tested again?
- Where else have you seen sensors in daily life?
